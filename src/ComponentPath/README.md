
コンポーネント内のパスの管理を行う

パス
　UI、状態管理クラスで使用する全パス

リストパス
　テンプレート内のforブロックに使われるパスをリストとする
　テンプレートを解析するだけで取得できる
　子要素とバインドした場合、子要素のリストパスを伝播させる必要がある
　リストインデックスの再構築で効率よく再構築できる

リスト要素パス
　上記リストで取得したパスに".*"を付与したもの
　リストパス登録時に同時追加すればよい

トラッキング監視用パス
　派生状態のgetterを定義しているパス
　状態管理クラスのprototypeから取得できる

備考
状態クラスのprototypeをたどるとgetterを取得できる
状態クラスのオブジェクトでObject.keys()を使えば、メンバーを取得できる

全パスの取得の仕方
静的な追加
　UIで出現するパスを求める
　そのパスの構造的な祖先パスも全パスに含める
　状態クラス、状態クラスのオブジェクトからは取得しない
動的な追加
　派生状態計算時、状態更新時アクセスのあったパス
UIから取得できないケース
　子コンポーネントで使用するパス
　　子コンポーネントの全パスを求めて、親子パス変換を行い親コンポーネントの全パスに含めるようにする

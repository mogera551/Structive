# 状態管理クラス
構造をトレースすれば、構造パスベースの状態ツリーを作れるのでは？
状態ツリーと状態プロキシーの統合
その際、リストインデックスの管理も統合する
　
# 状態にアクセスする可能性がある場面
RO　派生状態の計算
RW　状態更新時（イベント）
RW　状態更新時（ノードプロパティ更新）
RO　初期化時
RW　接続時（connectedCallback）
RW　解除時（disconnectedCallback）
RO　状態更新後、接続時後のレンダリング

# 用語
プレーンなパス
　ドットを含まないパス

# 処理の検討
## 初期化
UIで使用する全パスの取得
　全て取得できるわけではない
　　派生状態の計算中や状態更新時参照するパスはアクセスされるまで分からない
　　子コンポーネントで使用するパスを子コンポーネントの報告がないと分からない
　構造パス解析の累積パスを使って、親の構造パスも取得する
    

状態の初期化
const state = Reflect.constructor(StateClass);

状態ツリーの作成（staticDependencyに従い）
stateのプレーンなパスからツリー計算を開始する
